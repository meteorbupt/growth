> 拆解 AI 究竟是"一小撮人的自嗨"还是"百年一遇的技术变革"，理清关于 AI 我们高估了什么、低估了什么，以及在 Agent 时代个人如何重新定位——核心结论：旧地图已经失效，"执行"被外包之后，定义"什么是好"的品味与判断力才是不可替代的能力。

## 核心内容

### 一、是"自嗨"还是"百年变革"？——先扔掉旧地图

Cage 的判断不是"AI 很热所以重要"，而是从**"衡量对象变了"**这一根本点切入。他认为这是一次平台级的范式迁移，每一次平台迁移的规律都是：**新范式原生生长的公司赢，旧范式渐进迁移的公司输。**

**一个极具说服力的类比：**

> "用 DAU 这样的互联网指标给 AI 公司估值，就像 1999 年用页面停留时间给 Google 估值。"

Yahoo 用户停留 15 分钟，Google 用户 30 秒就离开——那时的分析师会说 Yahoo 胜出。但 Google 用户离开得越快，恰恰说明效率越高、商业意图识别得越好。**过去 30 年互联网的 DAU / MAU、To B / To C 分析框架，在 Agent 时代正在被颠覆，因为衡量对象变了。**

**用真金白银佐证"变革已经发生"：**

- Anthropic 的 ARR（年化收入）从 2025 年底约 90 亿美金，到 2026 年 2 月冲到 190 亿、3 月达到 300 亿，同比增长约 **1400%**。
- 听说过 Claude 的用户规模不到 ChatGPT 的 1%，但这 1% 是 token 消耗最密集、任务价值最高的用户。**"高 ARR 和低 DAU 可以同时成立"**——因为价值集中在头部任务，不在用户规模。

**对"是不是自嗨"的直接反驳——硅谷的体感已经变了：**

2025 年的硅谷技术变化还偏线性、甚至有人觉得停滞；但 2026 年春天体感完全不同——"几乎所有 AI 工程师和研究员都在过去一两个月从写代码变成只做 review，很多人直接承认 Claude Code 比自己强"。这是实打实的**"Move 37 时刻"**。

> 💡 关键认知：判断 AI 是不是真变革，不要看"有多少人在聊"（热度），要看**价值公式和衡量尺度是否被颠覆**（结构性）。

### 二、关于 AI，我们高估了什么、低估了什么？

这是全篇最有"认知刷新感"的部分。

#### 高估的部分（被旧框架误导的预期）

1. **高估了"用户规模 / 流量"的价值。**
   互联网时代 `DAU × ARPU` 的公式已经失效。Claude Code 用户规模只有 ChatGPT 的约 1/20，但两者 ARR 已基本持平甚至反超。"一个以工程师为核心用户、垂直扎进 Coding 的产品，用 5% 的用户量跑出可比收入，在互联网时代是不成立的。"

2. **高估了 To B / To C 这个分类本身。**
   他认为这个二分法"可能就是互联网时代的一个惯性"。增长最快的 AI 产品（Claude Code、Cursor、Perplexity、Manus）几乎都是 **Prosumer**（工程师、创意工作者、小企业主）先自发采用、再自下而上渗透到组织。"工程师愿意用个人信用卡买 Claude Code 做公司项目，这算 To C 还是 To B？"——**To B / To C 的边界已经模糊到无法分类。**

3. **高估了单点应用、对话式 Chat 的天花板。**
   他把 ChatGPT 验证的 chat 场景比作"今日头条"（信息分发），而把 Coding Agent 比作"抖音"级别的机会——后者通过多轮、长程任务把智能的刺激持续放大，是信息分发之后的下一次跃迁。

#### 低估的部分

1. **低估了"AI 加速 AI 自己"的飞轮速度。**
   这是 Cage 反复强调的、进入 2026 年最重要的结构性变化：**"AI 第一次开始系统性地加速 AI 自己。"** Anthropic 在 2026 年 2–3 月的 52 天里发了 70 多个产品 features，"很多是 Agent 写、Agent 测试、Agent 部署的"；OpenAI 发布 Codex 时用 "no manual code" 描述这种工作模式。过去模型格局以 6 个月为周期切换，现在竞赛窗口被压缩到以月、甚至以周为单位。

2. **低估了"高价值任务"对白领工资池的冲击。**
   **"Coding is eating SaaS, and the white-collar is next."** 他把哲学问题摆上台面：**"如果 execution 都被 AI 吃掉了，人该做什么？"** 共同点为"高智力门槛、单位智能消耗高、单位产出价值高"的工种（法律、金融、Coding、程序员）会先被冲击。全球白领工资总额约 **18–20 万亿美金**，这是 Agent 替代的"终极预算池"——而 IT 预算和人力成本预算的差距是 **1–2 个数量级**。

3. **低估了"Agent 作为新物种加入经济系统"的彻底性。**
   他提出新坐标系 **To Human / To Agent**：Agent 本身成为生产者和消费者，自主搜索、调用 API、开启 runtime、做采购决策、完成支付。**"生产者和消费者同时是非人类的双边市场，这在人类经济史上从未出现过。"**

### 三、Agent 时代：AI 系统性加速自己，价值公式重写

#### 1. Agent 不是 Chat 的升级，是"新物种"

分水岭是 **Opus 4.5**（被他比作"GPT-3 到 GPT-4 的跨代际模型"），越过"Agentic 能力拐点"后，**"越薄的 Harness 反而越强"**——Claude Code 核心的 Agent Loop 只有几十行代码，配套工程（上下文压缩、Multi-Agent 协调、工具调用）才是 Harness 的真正厚度。这意味着：**做中间层的、靠重工程"包住不确定性"的公司，压力会越来越大。**

#### 2. 价值公式：从"卖 Token"走向"卖 OS / 云平台"

模型公司的商业模式正在从卖 Token 扩展向卖 OS / 云平台那样的生态。一旦 Agent 的定义、状态、记忆全部托管在一家模型公司，切换成本就是整个 AI workflow 的重建——**"就像 AWS 的粘性从来不是 EC2 本身，是 IAM + VPC + S3 + Lambda 编织成的一张状态网。"** 换句话说，LLM 公司第一次有了护城河。

模型公司的终局形态是**争夺"新一代 OS"的位置**（参照 OS 格局：Android ~46%、Windows ~30%、iOS ~20%）。

#### 3. 付费逻辑：从 per-seat 走向 per-outcome

- Decagon 的 per-resolution 模型：只有 AI 真正解决了问题才收费。
- Sierra 直接把 "pay for a job well done" 写进产品。
- 传统 SaaS 跟着企业 IT 预算走；AI-native vertical agent 瞄准的是**人力成本池**。

> 新价值公式：`任务价值 × 完成率 × take rate`，终点是"替代的人力成本"。旧公式：`流量 × 转化 × ARPU`，终点是广告费 / 订阅费。

#### 4. 软件正在被"重写"

软件从一开始就是 To Human 的（GUI、每一层抽象都是为了降低人的认知负担），而 Agent 不需要这些。一场系统性的**格式迁移**正在发生：从"为人类易读"迁移到"为 Agent 易操作"。

> Claude Code 选择 CLI 而不是 IDE 是早期信号：**"CLI 就是 Agent 的母语，bash is everything for coding agents。"** 未来的软件大概率是 `Model + Agent Harness + 按需生成的人类审阅层`。

#### 5. 范式转移的残酷规律

每一次技术平台迁移，旧范式领先者都拥有数量级更大的用户、更强的品牌、更厚的现金，但最终赢家几乎都是从零开始、为新范式重新设计的公司——**因为旧范式难免被之前的 best practice 牵绊，而没有成功经验可以跨范式持续成功。**（雅虎不是不知道搜索重要，是它的编辑导航逻辑和 Google 的爬虫算法在产品 DNA 层面互斥。）

> 甚至认为 **OpenAI 的 8 亿用户是"战略包袱"而非"战略资产"**——每一个面向轻度对话用户的优化，都是对 Agent 深水区的一次妥协。

#### 6. 看好的三个方向（Agent Harness 四层结构）

| 层 | 机会 | 说明 |
|----|------|------|
| **Runtime 层** | Agent Infra | 让 Agent 跑得更 scalable：需要 per-agent 状态隔离、Fork/Snapshot（分支探索）、Durable Execution（跨故障持久执行），传统为人类设计的 DB / 虚拟机要被重做 |
| **Context 层** | Vertical Harness | 在医疗、法律、金融把行业 know-how 做到极致（模型公司做不完的深度） |
| **Orchestration 层** | Agent 一等公民基建 | Agent Identity（"这个 Agent 是谁、谁授权、出了事找谁"）+ Agent Payment（Stripe 的 Machine Payments Protocol 之上做"Agent 版的 Plaid"） |

### 四、LLM 作为推理引擎，Coding 是最佳应用

Cage 一以贯之的技术判断：

> **"LLM 作为推理引擎，coding 是最好的应用场景：代码的逻辑比自然语言更清晰，执行的结果能由 AI 自动化验证。"**

**为什么 Coding 是 AGI 的最好环境（四点）：**

1. 数字世界的任务几乎都可以用 Coding 表达，做好 Coding Agent ≈ 白领工作大规模自动化可在现有方法论下实现；
2. Coding 让 AI Labs 自己直接受益（产品侧和模型开发都加速）；
3. "知识工作大规模自动化"已成为 AI Labs 内部竞争工具——**没有领先的 Coding model，相当于没有领先的 GPU**；
4. Coding 具备天然飞轮——**"feedback loop 是所有 AI 任务中最短、最清晰的。"**

> 结论性判断：**"Coding Agent 一旦真正跑通，AGI 的 90% 已基本实现。"**

**Coding 的四象限图谱**（X 轴=用户开发能力，Y 轴=需要 human-in-the-loop 的程度）：

- 左下 **Copilot for pro dev**：Cursor、Codeium / Windsurf
- 右上 **Agent for citizen dev**：Vercel V0、bolt.new、Replit agent——类比"GUI 出现前的命令行时代"，期待"施乐实验室式的 GUI 时刻"
- 重要预言：**"Task engine 会把市场从 5000 万开发者拓展到 5 亿知识工作者"**；下一代浏览器地址栏输入的不再是网址而是自然语言 prompt——"这里可能孕育着 AI 时代 Google 的机会"。

### 五、上下文工程（Context Engineering）：Agent 的核心胜负手

> "如果把 LLM 类比为计算机的 CPU，那么 context window 就是计算机的 RAM。"

Context Engineering 由 Andrej Karpathy 提出（"为下一步给 LLM 的上下文窗口填入刚刚好的信息"），它**覆盖并超越了 prompt engineering 和 RAG**，直指 Agent 开发的核心痛点。

**核心痛点：context decay（上下文衰减）**——随着上下文变长，模型注意力分散、推理能力下降。Manus 一个典型任务平均约 50 次工具调用，生产级 Agent 甚至可达数百次，不做 context 工程单次可能消耗 50 万 token、成本 1–2 美元。

**五大策略：**

| 策略 | 含义 |
|------|------|
| 转移 Offload | 把不必要的信息移出上下文 |
| 压缩 Reduce | 对冗长内容做摘要 |
| 检索 Retrieve | 按需拉取而非全量塞入 |
| 隔离 Isolate | 不同子任务用独立上下文 |
| 缓存 Cache | 复用稳定片段降低成本 |

### 六、AI 时代什么让你不可替代？——品味与判断力

这是节目标题点出的落点（也是单独剪出的高赞切片《那些真正与人有关的事，正在变得越来越重要》），也是对"个人出路"的回答。

**Cage 的判断：当 execution 被外包，剩下的是"定义什么是好"的能力——也就是 Taste（品味 / 判断力）。**

- 在 Multi-agents 里，真正值钱的不是 execution agent，而是 **planning agent**（定角色、开 agent、派任务、做裁决的核心）。**当"怎么做"被外包，"做什么"和"判断什么是好"成为新的稀缺。**

- 拾象体系对品味的拆解：**Taste = 判断力（Judgment）+ 构思力（Ideation）**，可还原为"比较（Compare）+ 延伸（Extend）"。当"做什么 / 怎么做"都能交给 AI，**"定义目标函数：什么是更好"** 仍然依赖人类——"技能可以复制，流程可以自动化，但判断标准必须被'选择'。"

- 一个反直觉的洞察：**"人类的品味本质上是一个长期训练出来的'隐式模型'，区别在于——AI 在系统性训练品味，而人类大多还停留在随机形成品味。"**

- 关于就业冲击的冷峻观察：**"美国社会是一个梨形结构，中产占比极高，当他们真的被替代之后，社会结构会往哪个方向走，会是一件很不稳定的事。"**——不回避冲击，但把个人的出路指向"品味 / 判断力 / 规划能力"，而非"更努力地卷技能"。

## 给个人的实践启示

> 落到行动上，把"认知"变成"做法"。

1. **先获得"一手体感"，别停在二手信息。** 反复强调要"对模型边界有一手体感"——真正把最强的模型（Claude Code、Cursor 等）用起来、自己跑任务，而不是只看新闻和测评。

2. **从"要答案"切换到"做比较 / 做选择"。** 一次生成多个候选，强制自己做选择并追问"为什么选这个、标准是什么"——这既训练判断力，又给 AI 高质量反馈，形成 `人类标准 → AI 生成 → 人类判断 → 标准强化` 的闭环。

3. **从"模糊需求"切换到"显式标准"。** 不要说"帮我写一篇不错的文章"，而要明确"什么叫好（结构 / 风格 / 信息密度）、优先级、参考标准"。**你提供的不是任务，而是"判断函数（evaluation function）"——而定义判断函数正是人不可外包的部分。**

4. **从"执行者"转向"规划者 / Planner"。** 把精力从"亲手做"迁移到"定方向、定角色、派任务、做裁决"。

5. **系统性地训练自己的"品味模型"。** 有意识地"看大量样本 + 做优劣比较 + 内化判断标准"，把被动的经验积累变成主动的判断系统构建——因为 AI 已经在这么做了。

6. **不要被旧地图困住。** 无论求职、择业还是创业，不再用 DAU、To B / To C、per-seat 这些旧尺度衡量机会；改用"是否承载高价值任务、是否 agent-native、是否分到 80% 的 spending"来重新判断方向。

7. **重新看待"真正和人有关的事"。** 当 execution 被自动化，那些"决定生成什么、保留什么、放弃什么"的事，以及人与人之间真实的连接、判断、责任，反而会变得更稀缺、更重要。**"决定生成什么、保留什么、放弃什么的人，才真正拥有控制权。"**

## 总结反思

### 关键收获

- **认知刷新**：判断 AI 是不是真变革，看的是"衡量尺度是否被颠覆"而非"热度"。旧地图（DAU / To B-To C / per-seat）已失效，新坐标是 To Human / To Agent，新公式是 `任务价值 × 完成率 × take rate`。
- **结构性机会**：Agent 是"新物种加入经济系统"，Runtime（Agent Infra）、Context（Vertical Harness）、Orchestration（Identity / Payment）三层是被重写的增量。
- **个人定位**：在 execution 被外包的时代，稀缺的是"定义什么是好"的品味与判断力——要从"执行者"转向"规划者"，并系统训练自己的"判断函数"。
